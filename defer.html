How defer Works:

  1. The browser starts downloading app.js in parallel with parsing the HTML.
  2. The script execution is deferred until the HTML document is fully parsed.
  3. If multiple scripts have defer, they are executed in order as they appear in the HTML.

Why use defer?

  • Improves page load speed: The browser doesn't block rendering while downloading the script.
  • Ensures DOM is ready: The script executes after the full HTML is parsed, so document.querySelector() and other DOM manipulations wonâ€™t fail.
  • Preserves script order: Unlike async, defer ensures scripts run in the order they appear.

Summary

  • Use <\script defer> when the script interacts with the DOM.
  • Use <\script async> for independent scripts like analytics.
  • Use either defer or async, don't leave it blank
